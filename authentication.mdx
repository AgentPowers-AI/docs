## Overview

AgentPowers uses [Clerk](https://clerk.com) for authentication. All authenticated API requests require a Bearer token in the `Authorization` header.

## API Authentication

Include a Bearer token in the `Authorization` header:

```bash
curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://api.agentpowers.ai/v1/purchases
```

## CLI Authentication

The CLI handles authentication automatically:

```bash
ap login
```

This opens your browser for OAuth sign-in via Clerk. After authenticating, the CLI stores a long-lived token locally with restricted file permissions.

```bash
ap whoami    # Check current session
ap logout    # Clear stored token
```

### CLI Token Lifecycle

| Step | What happens |
|------|-------------|
| `ap login` | Browser OAuth via Clerk, long-lived token stored locally |
| API calls | CLI sends token automatically on every request |
| `ap logout` | Token revoked server-side, local credentials deleted |
| Token expiry | After 90 days, re-run `ap login` to get a new token |
| New login | Creating a new token automatically revokes the previous one |

## Plugin Authentication

The MCP plugin shares the same auth token stored by the CLI. Log in once with the CLI and the plugin picks it up automatically.

## Test Accounts

Accounts flagged `is_test_user = true` are sandbox accounts (used by QA tooling and internal
testing) that are prevented from triggering real-world side effects. The flag is returned on
`GET /v1/auth/me` so callers can verify their account state.

For a flagged account, the API:

- Returns `403` on `POST /v1/checkout` â€” no Stripe Checkout session is created.
- Returns `403` on `POST /v1/sellers/connect` â€” no Stripe Connect account is onboarded.
- Skips transactional email sends (receipts, sale notifications, payout confirmations,
  skill-published notices). The endpoint returns success to the caller, but no email leaves the system.

Skill slugs with the prefix `qa-sandbox-` are also hidden from the public `GET /v1/search`
endpoint so QA-published skills never surface to real buyers. Such slugs remain directly
fetchable via `GET /v1/skills/{slug}` for tooling that needs to inspect them.

The flag is set manually by platform operators on the `users` table; there is no public API
to toggle it.

## Rate Limits

| Tier | Limit |
|------|-------|
| Public reads | 60 requests/minute |
| Authenticated reads | 10 requests/minute |
| Write operations | 20 requests/minute |

Rate limits are per IP address for all requests. Exceeding the limit returns `429 Too Many Requests`.
