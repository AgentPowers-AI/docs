## Overview

AgentPowers uses [Clerk](https://clerk.com) for authentication. The API supports three types of Bearer tokens, checked in order:

1. **CLI long-lived tokens** (`ap_cli_...`) -- 90-day opaque tokens for CLI/plugin use
2. **Dev bypass tokens** (`dev_...`) -- Development mode only, never available in production
3. **Clerk JWTs** -- Short-lived tokens issued by Clerk for web/frontend use

## API Authentication

Include a Bearer token in the `Authorization` header:

```bash
curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://api.agentpowers.ai/v1/purchases
```

## CLI Authentication

The CLI handles authentication automatically:

```bash
ap login
```

This opens your browser for OAuth sign-in via Clerk. After authenticating, the CLI exchanges the short-lived Clerk JWT for a **90-day CLI token** via `POST /v1/auth/cli-token`. The token is stored locally at `~/.agentpowers/auth.json` with restricted file permissions (0600).

```bash
ap whoami    # Check current session
ap logout    # Clear stored token
```

### CLI Token Lifecycle

| Step | What happens |
|------|-------------|
| `ap login` | Browser OAuth via Clerk, then JWT exchanged for 90-day `ap_cli_` token |
| API calls | CLI sends `Authorization: Bearer ap_cli_...` on every request |
| `ap logout` | Token revoked server-side via `POST /v1/auth/revoke-cli-token`, local file deleted |
| Token expiry | After 90 days, re-run `ap login` to get a new token |
| New login | Creating a new CLI token automatically revokes the previous one |

### Token Exchange Endpoint

Exchange a Clerk JWT for a long-lived CLI token:

```bash
curl -X POST "https://api.agentpowers.ai/v1/auth/cli-token" \
  -H "Authorization: Bearer CLERK_JWT"
```

Returns:

```json
{
  "token": "ap_cli_abc123...",
  "expires_at": "2026-06-01T00:00:00Z"
}
```

### Token Revocation Endpoint

Revoke a CLI token (used during logout):

```bash
curl -X POST "https://api.agentpowers.ai/v1/auth/revoke-cli-token" \
  -H "Authorization: Bearer ap_cli_abc123..."
```

## Plugin Authentication

The MCP plugin shares the same auth token stored by the CLI at `~/.agentpowers/auth.json`. Login once with the CLI and the plugin picks it up automatically.

## Admin Endpoints

Admin endpoints require the `is_admin` flag on the user record. Non-admin users receive `403 Forbidden`. These endpoints are used for marketplace moderation:

| Endpoint | Description |
|----------|-------------|
| `GET /v1/admin/claims/sandbox` | List sandbox claims awaiting review |
| `POST /v1/admin/claims/{id}/approve` | Approve or reject a sandbox claim |
| `GET /v1/admin/abuse/flagged` | View flagged abuse reports |

Admin status is set at the database level and cannot be self-assigned.

## Rate Limits

| Tier | Limit |
|------|-------|
| Public reads | 60 requests/minute |
| Authenticated reads | 10 requests/minute |
| Write operations | 20 requests/minute |

Rate limits are per IP for unauthenticated requests and per user for authenticated requests. Exceeding the limit returns `429 Too Many Requests`.
